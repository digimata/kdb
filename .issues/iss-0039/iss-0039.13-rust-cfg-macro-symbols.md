# ISS-0039.13 :: Rust symbols inside cfg-macro / token-tree wrappers not extracted

## Summary

Symbols defined inside `cfg!` attribute macro blocks or other token-tree wrappers are not extracted by the symbol extractor. Tree-sitter parses the macro invocation but doesn't expand it, so items inside the token tree are invisible.

## Example

```rust
cfg_if::cfg_if! {
    if #[cfg(unix)] {
        pub struct TcpStream { ... }
        impl TcpStream { ... }
    } else {
        pub struct TcpStream { ... }
        impl TcpStream { ... }
    }
}
```

`TcpStream` is defined inside a `cfg_if!` macro block. Tree-sitter sees the macro invocation and its token tree, but doesn't parse the contents as Rust items. The symbol extractor never finds `TcpStream`.

## Evidence

Tested against tokio: `TcpStream` "symbol not found" in `stream.rs` — the declaration is inside a macro block.

## Root cause

Tree-sitter fundamentally cannot expand macros. The token tree contents are opaque. This is the same limitation as R8 (macro-generated usage) but for symbol *definitions* rather than usages.

## Fix direction options

1. **Heuristic parsing of token trees**: Scan the token tree content as if it were normal Rust source. Would catch `cfg_if!` and similar structural macros but would be fragile for procedural macros.

2. **`cargo expand` integration**: Run `cargo expand` to get fully expanded source, then extract symbols from that. Heavy dependency, slow, but accurate.

3. **Accept the gap**: Document that symbols inside macro blocks require the symbol to also be defined outside a macro in at least one cfg branch. This is a tree-sitter fundamental limitation.

Recommendation: option 1 for common patterns (`cfg_if!`, `cfg_attr`), option 3 for everything else.
