# ISS-0039.10 :: TSX/JSX identifiers broken in real code

## Summary

T11 (JSX component identifier) passes in the eval suite but fails on real TSX codebases. Multiple JSX identifier patterns are not being resolved in sonner.

## Affected patterns

1. **JSX tag identifier**: `<Loader />` — `Loader` not found as a usage
2. **JSX expression identifier**: `{… ?? CloseIcon}` — `CloseIcon` not found
3. **Identifier call inside JSX**: `{isAction(x)}` — `isAction` not found

## Root cause (suspected)

The T11 eval test likely covers only the simplest JSX case (`<Foo />`). Real TSX has identifiers in:
- JSX opening/closing tags (`<Foo>`, `</Foo>`)
- JSX self-closing tags (`<Foo />`)
- JSX expression containers (`{Foo}`, `{cond ? Foo : Bar}`)
- Function calls inside JSX expressions (`{isAction(x)}`)
- JSX attribute values (`prop={Foo}`)

The usage scanner's `is_usage_identifier` may be checking for `jsx_identifier` but the tree-sitter TSX grammar uses different node kinds for these different positions.

## Evidence

Tested against sonner (`~/Documents/repos/sonner`):
- `Loader` imported and used as `<Loader />` — not found
- `CloseIcon` imported and used in `{… ?? CloseIcon}` — not found
- `isAction` imported and called inside JSX — not found

## Fix direction

Audit all tree-sitter TSX node kinds that represent identifier usage in JSX positions. Likely need to add several node kinds beyond `jsx_identifier` to the usage scanner.

Key change site: `src/index/code_refs.rs` — `is_usage_identifier` for TS/TSX
