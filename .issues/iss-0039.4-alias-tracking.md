# ISS-0039.4 :: Aliased imports not resolved

## Summary

`refs -s` misses all usages of aliased imports across every language. The local alias name is stored in `ResolvedImport.names`, but `symbol_lookup` searches by the definition name. The two never match.

## Root cause

When a resolver encounters an aliased import (`as` rename), it stores the **local alias** in `ResolvedImport.names`:

```rust
// e.g. `use crate::foo::Bar as B;` produces:
ResolvedImport { names: vec!["B"], resolved_path: Some("foo.rs"), .. }
```

The usage scanner in `code_refs.rs` then finds identifier `B` in source and looks it up in `ImportedBindings`. The binding maps `"B" ‚Üí foo.rs`. So far so good.

But `symbol_lookup` searches `foo.rs` for a symbol named `"B"` ‚Äî which doesn't exist. The actual symbol is named `"Bar"`. The reference is silently dropped.

There's no alias‚Üídefinition mapping anywhere in the pipeline.

## Affected categories

| Test | Lang | Category | Example |
|---|---|---|---|
| R3 | Rust | Aliased import | `use crate::foo::Bar as B; B::new()` |
| T3 | TS/JS | Aliased import | `import { foo as f } from './bar'; f()` |
| P3 | Python | Aliased import | `from foo import bar as b; b()` |
| G2 | Go | Aliased import | `import p "pkg"; p.Foo()` |

## Fix direction

Add an alias map to `ResolvedImport`:

```rust
pub struct ResolvedImport {
    pub raw: String,
    pub resolved_path: Option<PathBuf>,
    pub kind: ImportKind,
    pub names: Vec<String>,
    pub aliases: HashMap<String, String>,  // local_name ‚Üí definition_name
    pub line: usize,
}
```

Each resolver populates `aliases` when it encounters an `as` rename. The usage scanner then uses `aliases.get(local_name)` to find the definition name for `symbol_lookup`.

Key change sites:
- `src/resolve/mod.rs` ‚Äî add `aliases` field to `ResolvedImport`
- `src/resolve/tsjs.rs` ‚Äî populate aliases for `import { foo as f }` patterns
- `src/resolve/rust.rs` ‚Äî populate aliases for `use Foo as Bar` patterns
- `src/resolve/python.rs` ‚Äî populate aliases for `import foo as f` / `from foo import bar as b`
- `src/resolve/go.rs` ‚Äî populate aliases for `import p "pkg"` (alias is the package name)
- `src/index/code_refs.rs` ‚Äî use alias map in `symbol_lookup`
