# ISS-0039.9 :: Go same-package references not tracked

## Summary

In Go, all `.go` files in the same directory share a package namespace. Symbols defined in one file are accessible in other files of the same package without any import statement. `refs -s` currently only finds references through import resolution, so it misses all same-package usage.

## Root cause

`refs -s` builds references by: (1) resolving import statements to source files, (2) scanning for identifier usages that match imported names. Go same-package access has no import statement — files in the same directory implicitly share scope.

## Repro

```
// pkg/kubelet/kubelet.go
type Kubelet struct { ... }

// pkg/kubelet/kubelet_pods.go (same package, no import needed)
func (kl *Kubelet) GetActivePods() []*v1.Pod { ... }
```

`refs -s Kubelet` finds only the definition in `kubelet.go`, not the usage in `kubelet_pods.go`.

## Real-world impact

Tested against kubernetes: `Kubelet` struct is used extensively across ~20 files in `pkg/kubelet/`, all same-package. Only 5 files import it cross-package. kdb finds the definition only.

This is Go-specific — no other supported language has implicit cross-file scope sharing without imports.

## Fix direction

For Go files, add a same-package scan: when building references for a Go symbol, also scan all other `.go` files in the same directory for identifier usages matching the symbol name.

This is orthogonal to import resolution — it's a separate reference source that only applies to Go.

Key change site:
- `src/index/code_refs.rs` — after import-based reference building, add a Go-specific pass that scans sibling files in the same directory

### Complexity notes

- Need to determine package membership by directory (Go convention: one package per directory)
- `_test.go` files are in the same package but may use `package foo_test` (external test package) — need to handle both
- Performance: for large packages (kubernetes `pkg/kubelet/` has ~60 `.go` files), this means scanning all siblings for every symbol
