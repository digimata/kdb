# ISS-0039.1 :: Python `from X import name` symbol binding broken

## Summary

`refs -s` finds zero references for Python symbols imported via `from module import name`. This breaks the most common Python import pattern and means Python recall is 0%.

## Root cause

In `src/resolve/python.rs:178` (`push_from_import_statement`), `from target import foo` produces two `ResolvedImport` entries:

1. **Module-level**: `resolved_path = Some("target.py")`, `names = []` (empty)
2. **Name-level**: tries to resolve `foo` as a submodule (`target/foo.py`), fails, so `resolved_path = None`, `names = ["foo"]`

In `src/index/code_refs.rs:120` (`ImportedBindings::from_imports`), entry #1 is useless (empty names) and entry #2 is skipped (`resolved_path` is None). The binding `"foo" → target.py` is never created.

The resolver conflates "import a symbol from a module" with "import a submodule". When the submodule doesn't exist, the import is silently dropped.

## Repro

```rust
// tests/refs_eval.rs — python_p01_direct_import
eval_refs(
    &[
        ("target.py", "def foo():\n    pass\n"),
        ("caller.py", "from target import foo\nfoo()\n"),
    ],
    "target.py",
    "foo",
);
// Expected: 1 definition + 1 usage
// Actual:   1 definition + 0 usages
```

## Affected categories

| Test | Category | Description |
|---|---|---|
| P1 | Direct import | `from target import foo; foo()` |
| P6 | Relative import | `from .target import foo; foo()` |
| P7 | Decorator usage | `from target import decorator; @decorator` |
| P8 | Type annotation | `from target import Bar; def f(x: Bar)` |
| P9 | Class instantiation | `from target import Bar; Bar()` |

## Fix direction

When `from X import name` fails to resolve `name` as a submodule, fall back: if `X` itself resolved to a file, emit a `ResolvedImport` with `resolved_path = X's path` and `names = [name]`. This creates the binding the usage scanner needs.

Key change site: `src/resolve/python.rs:207-228` — after the `resolve_module_path` loop fails, check if the parent module resolved and use that path instead of None.
